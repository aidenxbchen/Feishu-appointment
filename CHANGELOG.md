# Changelog

## v1.3.1 — 姓名匹配修复 + 后台处理稳定性 *(2026-06-27)*

**Bug 修复**

- **中文姓名暴力映射问题**：修复外部联系人的中文姓名被错误映射到团队内部成员的问题。现在仅对纯拼音/英文/缩写输入才走 AI 归一化，含中文字符的姓名直接视为外部人员，跳过名单匹配
- **后台 worker 静默崩溃**：修复 `process_booking()` 中 `re` 模块未在顶层导入的问题（`import re` 仅写在了 `_sync_en_name_aliases()` 函数内部）。当飞书通讯录搜索不到预约人时，`re.search()` 抛出 `NameError`，被 worker 的 `except Exception: pass` 静默吞掉，导致预约卡在 `processing` 状态，日历和 IM 通知均不触发

## v1.2 — 按需同步 + 模糊姓名匹配 *(2026-06-24)*

**核心改动**

- **按需同步架构**：移除定时器，改为每次打开页面时同步飞书日历忙闲和团队名单，无后台常驻轮询
- **三层模糊姓名匹配**：
  1. 飞书通讯录直接搜索
  2. 本地别名表（大小写不敏感）
  3. Claude 兜底：携带完整团队名单 + 邮箱前缀推断
- **英文名 / 花名自动同步**：每次打开页面搜索飞书「个人说明书」文档，从标题解析英文名和花名写入 `member_aliases` 表
- **IM 通知修复**：修正 `lark-cli` 命令格式（`im +messages-send --user-id`）
- **日程标题优化**：使用解析后的显示名而非用户原始输入；AI 无法匹配时才保留原文

**前端改动**

- 页面标题改为「我想和你见一面」
- 打开页面时预加载忙闲数据，进入第二步无需等待
- 成功卡片正面文案：「我已经等不及见到你，我们 [日期时间] 见」
- 翻面提示改为明显的药丸状徽标

**数据**

- 新增 `member_aliases` 表，收录团队 20 名在职成员的英文名和花名映射

## v1.1 — 异步预约流水线 + SQLite 缓存 *(2026-06-24)*

**核心改动**

- **SQLite 本地数据库**（WAL 模式）：新增 `bookings`、`busy_slots`、`cache_meta` 三张表
- **后台 Worker**：独立线程每 5 分钟同步飞书忙闲数据，异步处理待完成预约
- **前端立即返回**：提交后前端直接显示成功，无需等待飞书 API
- **完整预约流水线**（后台执行）：姓名查询 → 创建日程 → 邀请参与者 → 发送 IM 通知
- **防重复预约**：线程锁 + 数据库原子查询双重保障
- **自动重试**：失败预约最多重试 3 次
- **外部成员处理**：跨租户用户静默跳过邀请，不报错

---

## v1.0 — 初始版本 *(2026-06-23)*

**功能**

- 飞书日历 freebusy 集成，实时查询空闲时段
- 三步预约流程：填写信息 → 选择时段 → 确认
- AI 日程标题生成（Claude CLI）
- 飞书通讯录姓名查询，自动添加参会人
- 工作日过滤，展示未来 14 天可预约时段
- 简洁移动端 UI，零第三方依赖

**技术栈**

- Python 3 + Flask，同步 API（飞书调用在请求中内联执行）
- 原生 HTML/CSS/JS 单文件前端
- Nginx 反向代理 + systemd 进程管理
